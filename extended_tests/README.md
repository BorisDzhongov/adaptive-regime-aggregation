# ARA Extended Tests

This directory contains additional experimental analyses for the **Adaptive Regime Aggregation (ARA)** framework.

These tests are **separate from the reference experiments used in the Zenodo paper** and serve as an experimental environment for exploring the behaviour of the ARA architecture under a wider range of scenarios.

The experiments are intended for:

- stress testing the ARA architecture
- sensitivity analysis of parameters and regimes
- exploratory comparison with alternative aggregation methods
- investigating behaviour under structured uncertainty

Results in this directory should therefore be interpreted as **exploratory or architectural analyses**, not as part of the official reproducibility package of the paper.

# Experimental Philosophy

The experiments in this directory are designed to explore **architectural properties of ARA** rather than to claim universal superiority over existing MCDA methods.

In particular, some tests intentionally examine situations where:

- information is incomplete
- subsystems have heterogeneous reliability
- signals conflict or contain noise
- adaptive coordination may be beneficial

Because of this, some benchmarks may involve **asymmetric information structures**, where ARA has access to signals (e.g. reliability indicators) that classical aggregation baselines do not use.

Such tests should therefore be interpreted as **architectural stress tests**, not as equal-information competitions between methods.

---

# Test Categories

## Stress Tests

Stress tests examine how ARA behaves under extreme or adversarial conditions.

Typical scenarios include:

- strong signal conflicts
- heavy noise
- missing observations
- subsystem shocks

The goal is to evaluate **robustness and stability of the coordination mechanism**.

---

## Sensitivity Analysis

Sensitivity experiments explore how ARA behaves when key parameters vary, such as:

- regime parameters (φ family)
- anchor influence
- subsystem reliability weights
- noise levels

These tests help understand **parameter stability and regime behaviour**.

---

## Comparative Experiments

Some experiments compare ARA with classical aggregation approaches such as:

- mean aggregation
- WSM
- TOPSIS
- PROMETHEE
- ELECTRE

These comparisons should be interpreted as **exploratory benchmarks**, not as definitive performance rankings.

The purpose is to illustrate how ARA behaves relative to standard aggregation mechanisms in different informational environments.

---

# Reliability Benchmark (Test E)

Test E examines scenarios where **subsystem reliability is structurally relevant**.

In these experiments:

- ARA receives local reliability signals (`Rg`, `Ri`)
- classical MCDA baselines operate on pooled observed scores

The benchmark is therefore intentionally **methodologically asymmetric**.

Its purpose is not to demonstrate universal superiority of ARA under equal information, but to evaluate whether **reliability-aware adaptive coordination provides advantages when reliability information is available and relevant**.

The test includes several scenarios:

- balanced reliable subsystems (control scenario)
- reliable vs noisy subsystem
- reliable vs biased subsystem
- complementary local reliability

Additional methodological notes are provided in the inline documentation of the test script.

---

# Reproducibility

All experiments are implemented as standalone Python scripts and can be executed independently.

Outputs are written to:

extended_tests/results/csv/

The experiments rely only on standard scientific Python libraries (primarily `numpy`) to ensure easy reproducibility.

---

# Relationship to the Paper

The experiments in this directory are **not required to reproduce the results of the Zenodo paper**.

They serve as an **open research environment** for exploring additional properties of the ARA architecture beyond the examples presented in the manuscript.
